[译]移动开发工具(移动开发2)

原文: The Mobile Developer’s Toolkit (Mobile Web Part 2)

和原文有删减，文中的有些网址需要翻墙。如翻译有误，请与我联系。谢谢！

在你想成为一个合格的移动开发者之前你必须熟练应用一些移动开发的工具。虽然你认为自己在某款智能手机的平台下开发很熟练，一个不切实际的想法是你需要去购买其它智能手机用来测试。所以接下来你应该有一个更好的准备：一个SDK。
但一个SDK只是工具之一，以下链接可以让你获得更多!

##SDKs, 模拟器

短时间内成功拥有一个真正的多手机模拟终端的最快的方式是有一个模拟器。而现如今：所有的智能手机都有一个可供你使用的SDK！

iPhone SDK（仅限Mac OS X） - 典型的SDK为典型的智能手机。包括iPhone模拟器。
Android的SDK（Mac OS X，Windows和Linux）
Firefox移动模拟器（又名非洲狐） - 火狐手机版正在开发新的诺基亚手机和soon Androids。
Opera移动模拟器 - 这是目前最快的模拟器，下载后无需安装可直接运行。
Opera Mini模拟器（基于浏览器的Java小程序） - 在浏览器模拟器。请注意，Opera Mini和Opera Mobile不同。

有关更完整列表，请参阅 Maximiliano Firtman’s 的手机模拟器：终极指南

##实验室设备
这些服务允许您实验室的实际设备测试。

perfectomobile

Keynote MITE（移动交互式测试环境）（仅限于Windows） - 一个华而不实的用户代理切换。没有真正的设备测试。

DeviceAnywhere（基于Java的软件，在Windows和Mac OSX上运行） - 这个服务让你仿佛在世界各地的实验室用实际的移动设备测试。唯一的缺陷是有点笨拙，但在市场上没有真正的替代品。如果你打算支持的不仅是智能手机。需要订阅。

##用户代理切换器

许多网站探测浏览器的用户代理，以检测它是否是一个移动设备。作为一个开发者这是一个挑战，因为很多开发是在桌面浏览器中完成的。随着用户代理切换器，开发者可以伪装自己的桌面浏览器作为手机浏览器。

Firefox的插件：用户代理切换器
Safari：打开开发工具栏（点击Safari浏览器 - >设置 - >高级），并单击开发 - >用户代理。选择一个预定义的用户代理或选择输入自定义代理“其他…”

Chrome浏览器：最简单的实现办法是设置命令行标志：（请注意，用户代理切换器扩展Chrome浏览器无法正常工作）

在Windows下更改Chrome的用户代理：

1.创建Chrome快捷方式副本。
用鼠标右键单击该副本，然后选择属性。
在目标领域添加–用户代理=“myagent”
例如目标：“C:\Documents and Settings\Username\Local Settings\Application Data\Google\Chrome\Application\chrome.exe” ––user-agent=”Mozilla/4.0 （兼容; MSIE 6.0; Windows NT的5.1）”

在Mac OSX下更改Chrome的用户代理：

打开终端
在终端中输入：/Applications/Google Chrome.app/Contents/MacOS/Google Chrome ––user-agent=”myagent”
例如命令：/Applications/Google\ Chrome.app/Contents/MacOS/Google Chrome ––user-agent=”Mozilla/4.0 (compatible; MSIE 6.0; Windows NT 5.1)”

移动检测
在服务器端，移动检测主要是由用户代理嗅探（需要用户允许以上代理切换）完成的，但也有一些其它方法。

“精简版”用户代理检测：这些方法实现一个简单的服务器端功能（通常在PHP）检测常见的移动用户代理。有几个版本，主要是基于由Andy Moore。有一个在WordPress移动包。

WURFL（无线通用资源文件） - 一个已经使用10+年的项目并且目前发展很好。而且它是开源的，提供了很多关于设备方面有价值的信息。

DeviceAtlas - WURFL是一个商业版本
