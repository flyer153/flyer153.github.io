node.js包管理器

使用 npm 安装包的􏲒􏲣格式为:
npm [install/i] [package_name]

eg: 
npm install express 或 npm i express

本地模式和全局模式

􏳕本地模式: npm install􏲒􏲣

全局模式: npm [install/i] -g [package_name]

eg:
npm install -g supervisor

| 模式 | 可通过require使用 | 注册PATH |
| 本地 | yes | no |
| 全局 | no | yes |

使用场景：
􏳈本地模式：将某个包作为工程运行时的一部分
全局模式：要在􏲒􏲣命令行下使用

创建全局链接： 

npm link(windows不支持)

作用：把全局包当本地包使用(在本地包和全局包之间创建符号链接,因为本地包不支持require)

eg:
    npm install -g express(全局模式)
在工程目录运行:
    npm link express 
    ./node_modules/express -> /usr/local/lib/mode_modules/express
在mode_modules子目录可以看到一个指向安装到全局的包的符号链接

同时也可以将本地包当做全局使用
在包目录(即package.json目录)运行npm link

包的发布

创建一个 文件夹 ，运行 npm init   (创建一个标准的package.json,按照提示填一些必要的信息)
创建一个发布账号：npm adduser  
验证是否取得账号：npm whoami
在包目录(即package.json目录)运行 npm publish 发布包完成
更新包：package.json修改version版本信息，重新发布即可
取消发布： npm unpublish

调试

####命令行调试

执行：node debug debug.js 启动调试工具

####远程调试

执行：node --debug[=port] script.js
     node --debug-brk[=port] script.js

node–debug 启动调试服务器。默认端口：5858
修改端口： –debug=1234

脚本暂停执行等待客户端连接： –debug-brk

####使用Eclipse调试

由于平常用sublime比较多，故调试方法略过，有需要可以参考node.js开发指南page48中的详细介绍

####使用node-inspector调试

开源的在线调试工具

安装和使用：

安装：npm install -g node-inspector
连接要除错的脚本的调试服务器： node --debug-brk=5858 debug.js
node-inspector
在浏览器中打开http://127.0.0.1:8080/debug?port=5858

使用： 就像用chrome中的debug一样使用
