通过设置视窗宽度等于device-width（设备宽度），现在在移动端打开页面看起来就好得多，我们只是告诉它设备宽度为

##关于视窗标签的扩展：

除了内容宽度解决了我们最大的困扰外，视窗标签还有更多的可选项：

属性 详情
width 视窗宽度（或设备宽度）。如果未定义，则默认为桌面大小（在mobile Safari浏览器中980px）
height 视窗高度（或设备高度）。通常无须设置
initial-scale （0-10.0）倍数，用于设置页面显示之后的初始值，通常：如果需要设置，将它设置为1.0，值越大 = zoomed in(放大)，值越小= zoomed out(缩小)
minimum-scale （0-10.0）的最小倍数，用户可以zoomed out，默认在mobile safari下为0.25
maximum-scale （0-10.0）的最小倍数，用户可以zoomed in，默认在mobile safari下为1.6
user-scalable （是/否）是否允许用户缩放/缩小(zooming in/out)，默认在mobile safari下为是

##针对智能手机进行优化：

这个标签原本是用来识别移动AvantGo浏览器中的内容。现在它已经被用来识别移动web站点的通用标准。但是具体
支持meta标签的范围程度要视浏览器而定。
在此处的另一个标签是windows特有的meta标签，最终用来识别移动内容的另一种手段，但此标签的缺点是，必须
指定特定宽度，这就使视窗标签没那么灵活(见上文)，并且，这个标签还未得到普遍支持。

最后，你的站点就是针对移动优化过的，如果你的doctype是介于 XHTML-MP 或 WML两者之一。然而，这种情况非常
稀少，现代的浏览器已经很好的支持旧的HTML4.01 和 HTML5+。
(此信息部分来自于智能手机web开发)
