Hg 常用命令

公司使用hg作为代码管理工具，第一次接触这个工具，一开始使用非常不习惯。一段时间下来发现hg和git的使用有很多相似之处。
以下整理一些hg的安装配置以及常用的hg的使用命令，作为备忘。

####常用命令
hg clone +url 克隆仓库
hg branch 查看当先在哪个分支
hg branch +分支名 创建新分支
hg ci -m “注释” 提交本地修改
hg push –new-branch 提交分支
hg push -f 强推（慎用）
hg pull +url 拉别人的分支
hg merge 合并对方的代码到自己的分支中
hg head 查看提交的记录信息
hg outgoing 查看要推送的代码，但不执行推送操作
hg merge +分支名 合并分支
hg up 将代码修改更新default上
hg up -C 放弃本地未提交的更改
hg st | grep M 看改了哪些文件
hg branches|grep +名字（zb） 看所有的分支名
hg st 查看项目的一些修改状态
hg add 添加将要合并的文件
hg remove +文件名 删除文件。
hg resolve –mark 解决文件冲突
hg rollback 回退版本

大概常用的命令就这么多，后续如有需要会继续更新。。
